New features: Collaborate with ease

May 30th, 2014

Today, we are excited to introduce two new features focused on enhancing communication among businesses and security researchers: vulnerability report messaging and business collaboration. Because we are building a marketplace where businesses and security testers can freely communicate to improve web security, we are constantly working to improve and increase communication between testers and businesses. These new features are an important part in helping us achieve this goal.

Vulnerability Report Messaging

For some time, we have wanted to build a feature that allows for more businesses and testers to communicate and better clarify vulnerability reports. Now, all program owners have logs and messaging capabilities. The next time that you have a question about a vulnerability report submitted by an awesome tester, comment directly from your dashboard get immediate clarification from the tester.

Business Collaboration

Most businesses have more than one person who works to secure a web application. In addition to web developers who will build and implement patches, there may be a CTO or CEO who keeps an eye on web application security. Some businesses may even have an external security company managing their security that needs access to vulnerability reports. Our new collaboration feature enables multiple users to work together on the same bounty program.

To use this feature, login to your account and access program settings from your Dashboard. At the top of the menu on the left, select “Collaborators” and add the email addresses of anyone needing access to your organization’s reports. (Note: Collaborators will need to create CrowdCurity accounts for report access.)

Are there any communication features you would like to add to CrowdCurity? Feel free to reach out to us if you have any feedback or ideas.
